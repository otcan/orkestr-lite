One real run

The task changes. The operating loop stays.

This run uses a sourced research task because it forces the whole system to work: browser research, files, a phone follow-up, and a scheduled return to the same context.

01 · Connect

Bring your own Codex account.

Docker is already running. Connect Codex, optionally link your own WhatsApp account, and open one private workstation. No second bot account and no public hosted agent are required.

02 · Execute

Give Codex a real computer.

Codex works inside a persistent Ubuntu desk with Chromium, terminal access, and the shared workspace. Watch it live, pause it, or take control without breaking the conversation.

03 · Keep the output

The work stays usable.

The report is not trapped in a chat reply. Its Markdown and HTML files stay in the workstation, where you can inspect, download, continue editing, or send them to WhatsApp.

04 · Continue from your phone

WhatsApp is part of the same conversation.

Here, the user asks for a follow-up from WhatsApp. Codex keeps the existing research context, updates the report, and returns the actual Markdown file to the same chat.

05 · Return on schedule

Recurring work does not need another system.

A weekly source review is scheduled and run through the same queue. The result returns to the existing conversation and is mirrored to WhatsApp. Manual requests and timers do not create disconnected agent histories.

What Orkestr adds

Codex is the intelligence. Orkestr is the operating layer.

Orkestr does not replace Codex and it is not a workflow-specific bot. It packages the surrounding infrastructure into one local product: persistent hosting, flexible inputs, a graphical desk, WhatsApp routing, schedules, files, history, and human takeover.

Built for operational work

Use the same loop wherever consistency matters.

Outreach Research contacts, prepare follow-ups, and return drafts for review.
Job search Review opportunities, prepare materials, and revisit the pipeline.
Messaging Inspect unresolved threads and bring decisions back to the human.
Recurring checks Inspect sites, reports, queues, or repositories on a schedule.
